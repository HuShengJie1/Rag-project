# 全文表格样式审计结果

本文件记录本次“全文表格统一”开始前，对 [thesis/毕业论文.tex](/Users/user/毕设/rag-project/thesis/毕业论文.tex) 中全部 `table` 环境所做的样式审计结果。审计对象为正文中的 10 张 LaTeX 表格，不包含图片类 `figure` 环境。

## 一、总体统计

- 表格总数：10
- 使用 `booktabs`（`\toprule/\midrule/\bottomrule`）的表格数：10
- 使用 `\hline` 的表格数：0
- 使用 `\resizebox{\textwidth}{!}{...}` 的表格数：10
- 显式设置表内字号命令的表格数：0
- 显式设置 `\arraystretch` 的表格数：1
- `\caption` 在前、`\label` 在后的表格数：10
- 带“注：”说明的表格数：2

## 二、审计结论

统一前的主要问题并不在于表格内容风格冲突，而在于以下几个排版层面的不一致：

1. 全部表格都使用了固定 `\resizebox{\textwidth}{!}{...}`，导致窄表也被强制放大，宽表又被压缩，最终呈现出的实际字号不一致。
2. 大部分表格没有显式设置表内字号，只有少数表格在注释位置使用了 `\small`，导致视觉上缺乏统一的字号基准。
3. 只有本地对比表设置了 `\arraystretch`，其他表格保持默认行距，造成表内疏密不一致。
4. 注释虽然都采用“注：”的学术写法，但实现方式是手工内联，未形成统一模板。

从样式基础看，全文表格已经具备较稳定的主流框架，即：

- 居中排版；
- `\caption` 后接 `\label`；
- 使用 `booktabs`；
- 无竖线；
- 以宽表整体缩放的方式控制版面。

因此，本次统一工作以这一主流框架为基础，只收敛字号、行距、注释和缩放策略，不改变表内数据与语义。

## 四、对统一工作的直接启示

根据上述审计结果，统一工作的重点应集中在以下四项：

1. 为所有表格补充统一的表内字号基准。
2. 为所有表格补充统一的表内行距。
3. 将固定 `\resizebox` 改为“仅超宽时缩放”的策略，避免窄表被放大。
4. 为少量带注释的表格提取统一的注释写法，避免手工格式漂移。
