# 4.1 章本地对比验证方案

## 一、验证目标

本地验证的目标不是重新进行最终选型，而是在同一工程认证文档、同一页面条件下，对不同解析方式的输出差异进行可复核展示，从而为论文第 4.1 章补充“本地对比验证”证据。验证重点在于：

- 表格结构是否得到保留；
- 阅读顺序是否与原始页面一致；
- 标题层级与段落边界是否清晰；
- 解析结果是否足以支撑后续语义感知切块。

## 二、拟比较的解析方式

### 1. 较弱基线：`pdftotext -layout`

- 定位：作为纯文本抽取基线。
- 作用：用于展示在复杂表格、扫描件和标题混排场景下，简单文本抽取会产生何种结构损失。
- 输入：原始 PDF。
- 输出：按页导出的纯文本文件。
- 预期现象：
  - 对扫描件可能直接输出空文本；
  - 对矩阵表格和课程设置表会出现列结构打散、阅读顺序错位等问题。

### 2. 传统 pipeline 类解析方式：复用项目内已有 MinerU pipeline 输出

- 定位：作为较强但仍存在局限的传统 pipeline 类版面解析结果。
- 作用：用于展示相较于纯文本抽取，结构恢复已有明显提升，但在符号、公式、标题层级及复杂布局上仍可能出现错误。
- 输入：项目内现有 pipeline 解析产物。
- 输出：Markdown 与 `content_list.json`。
- 说明：
  - 本次任务优先复用项目内已有真实输出，不额外修改原实验目录。
  - 若后续需要，可再补充其他传统解析器，但本次不编造不存在的结果。

### 3. 最终方案：MinerU2.5 对应的 VLM/hybrid 输出

- 定位：论文中最终采用的解析方案。
- 作用：展示其在复杂结构理解、表格恢复、标题识别、阅读顺序与后续切块可用性方面的优势。
- 输入：项目内现有 VLM/hybrid 解析产物。
- 输出：Markdown 与 `content_list.json`。

## 三、拟比较的测试文档与页面

### 文档 A：`data/system_docs/大数据-2025级本科生培养方案-v13-724.pdf`

- 页面 2：
  - 现象：毕业要求对培养目标的支撑关系矩阵。
  - 关注点：矩阵行列是否完整、勾选符号是否正确、后续是否能够按“行/列语义单元”切块。
- 页面 4：
  - 现象：课程设置相关表格集中出现。
  - 关注点：多表格场景下的列结构保留、表头识别与阅读顺序一致性。

### 文档 B：`data/system_docs/附件1-10 上海海洋大学信息学院课程质量评价办法.pdf`

- 页面 1：
  - 现象：文档标题与一级条款混排。
  - 关注点：扫描件下标题识别、多级标题恢复及正文顺序是否合理。
- 页面 4：
  - 现象：表格与公式混排。
  - 关注点：表格结构、公式可读性及章节边界是否得到保留。

## 四、比较维度

本次对比不强行构造数值分数，优先采用基于真实结果的定性评价。核心维度如下：

1. 表格结构保留：是否能够保留行列关系、表头与单元格对应关系。
2. 阅读顺序一致性：解析文本的线性顺序是否与人工阅读原页时的顺序基本一致。
3. 标题层级保留：一级标题、二级标题与正文边界是否清晰。
4. 后续切块可用性：解析结果能否直接支撑 4.1.2 所述的精细化切块与元数据绑定。
5. 主要问题：概括最典型的结构损失或识别错误。

## 五、复现与材料生成路径

### 1. 复现脚本

- `experiment/4.1章/run_compare.sh`
  - 作为外层入口，统一调用 Python 脚本。
- `experiment/4.1章/run_compare.py`
  - 自动定位文档、复用既有 pipeline/VLM 输出；
  - 调用 `pdftotext -layout` 生成弱基线文本；
  - 抽取重点页面对应的页级片段；
  - 在 `experiment/4.1章/generated/` 下整理输出索引。

### 2. 结果整理

- `results_index.md`
  - 汇总不同解析方式的输出路径。
- `local_validation_summary.md`
  - 用中文学术书面语总结本地验证发现。
- `table_local_parser_compare.tex`
  - 生成可直接写入论文的 LaTeX 表格。
- `screenshot_suggestions.md`
  - 给出建议截图页面、截图区域与展示意图。

## 六、当前方案边界

- 当前最稳妥的本地对比路径为“纯文本抽取弱基线 + 现有 pipeline 输出 + 现有 MinerU2.5 输出”。
- 若本地环境后续能够稳定运行更多传统解析器，可扩展为三种以上方案对比；但在当前资料条件下，不宜超出仓库中已有结果进行推断。
- 因此，本次输出将严格基于项目内真实文档与真实解析产物生成，不补写不存在的实验数据。
